## üî∑ 1. MULTIMEDIA DEVELOPMENT TOOLS

### ‚ùñ Overview
- **Multimedia Development** is the process of creating the individual assets (text, audio, video, graphics) that will be used in a multimedia project.
- **Development Tools** are the specialized software programs used to create, edit, and manipulate these assets.
- Before you can *assemble* a project (Authoring), you must *create* the parts (Development).

---

### ‚ùñ 1. Classification of Development Tools

#### **A. Text Editing and Word Processing Tools**
- **Purpose**: To create, edit, and format textual content.
- **Features**:
    - **Formatting**: Fonts, sizes, colors, styles (Bold/Italic).
    - **Spell Check**: Automatic error correction.
    - **OCR (Optical Character Recognition)**: Converting scanned images of text into editable text.
- **Examples**:
    - **Simple**: Notepad (Plain text).
    - **Complex**: Microsoft Word (Rich text), Adobe InCopy (Professional publishing).

#### **B. Painting and Drawing Tools (Graphics)**
- **Bitmap (Raster) Editors**:
    - **Purpose**: Editing photos and detailed images.
    - **Data**: Grid of pixels.
    - **Pros**: Realistic color blending.
    - **Cons**: Loses quality when scaled up (Pixelation).
    - **Examples**: Adobe Photoshop, GIMP, Corel PaintShop Pro.
- **Vector Draw Programs**:
    - **Purpose**: Creating logos, icons, and illustrations.
    - **Data**: Mathematical formulas (Lines, Curves).
    - **Pros**: Scalable to any size without quality loss.
    - **Cons**: Not good for realistic photos.
    - **Examples**: Adobe Illustrator, CorelDRAW, Inkscape.

#### **C. 3D Modeling and Animation Tools**
- **Purpose**: Creating three-dimensional objects and scenes.
- **Features**:
    - **Modeling**: Building the shape (Mesh).
    - **Texturing**: Applying surfaces (Wood, Metal).
    - **Rendering**: Calculating light and shadow to create the final image.
- **Examples**: Blender (Free), Autodesk Maya (Movies), 3ds Max (Games).

#### **D. Image Editing Tools**
- **Purpose**: Enhancing and correcting digital images.
- **Features**:
    - **Filters**: Blur, Sharpen, Noise reduction.
    - **Layers**: Stacking images on top of each other.
    - **Masking**: Hiding parts of an image.
- **Examples**: Adobe Lightroom (Color correction), Snapseed (Mobile).

#### **E. Sound Editing Tools**
- **Purpose**: Recording, editing, and mixing audio.
- **Features**:
    - **Waveform View**: Visual representation of sound.
    - **Cutting/Splicing**: Removing mistakes.
    - **Effects**: Reverb, Echo, Equalization (EQ).
    - **Multi-track**: Mixing voice, music, and SFX together.
- **Examples**: Audacity (Free), Adobe Audition, Pro Tools (Industry Standard).

#### **F. Video Editing Tools**
- **Purpose**: Assembling video clips into a movie.
- **Features**:
    - **Timeline**: Arranging clips in order.
    - **Transitions**: Fades, Wipes between clips.
    - **Color Grading**: Changing the mood via color.
- **Examples**: Adobe Premiere Pro, Final Cut Pro (Mac), DaVinci Resolve.

#### **G. Animation and Visual Effects Tools**
- **Purpose**: Creating moving graphics.
- **Examples**: Adobe After Effects (VFX), Toon Boom Harmony (2D Cartoons).

---

### ‚ùñ Quick Revision Points

‚úì **Bitmap**: Photoshop (Pixels).
‚úì **Vector**: Illustrator (Math).
‚úì **3D**: Blender, Maya.
‚úì **Audio**: Audacity.
‚úì **Video**: Premiere Pro.

## üî∑ 2. MULTIMEDIA AUTHORING TOOLS & FEATURES

### ‚ùñ Overview
- **Authoring Tools** are the "Glue" that holds a multimedia project together.
- They allow you to **integrate** the assets created by development tools (images, sound, video) into a single interactive application.
- They provide the framework for **Navigation**, **Interactivity**, and **Logic**.

---

### ‚ùñ 1. Why Use Authoring Tools?
- **Integration**: Combines different media types.
- **Interactivity**: Makes the content respond to user actions (Clicks, Typing).
- **Ease of Use**: Allows non-programmers to create software using visual interfaces.
- **Efficiency**: Speeds up development compared to writing raw code (C++/Java).

---

### ‚ùñ 2. Key Features of Authoring Software

#### **A. Organizing and Editing Features**
- **Visual Flow**: Tools to visualize the structure (Flowcharts, Storyboards).
- **Asset Management**: A library to store and organize imported files.
- **Drag-and-Drop**: Placing elements on the screen easily.

#### **B. Programming and Scripting Features**
- **Visual Programming**: Connecting icons to create logic (No coding).
- **Scripting Languages**: Built-in languages for advanced control.
    - *Lingo* (Director)
    - *ActionScript* (Flash)
    - *JavaScript* (Web)
- **Benefits**: Allows complex logic (e.g., "If score > 10, play win sound").

#### **C. Interactivity Features**
- **Events**: Defining triggers (Mouse Click, Key Press, Time Limit).
- **Actions**: Defining responses (Jump to Scene 2, Play Video, Exit).
- **Hotspots**: Invisible clickable areas on an image.

#### **D. Playback and Debugging Features**
- **Preview**: Testing the application instantly without building it.
- **Debugger**: Finding errors in the code (Variables, Logic loops).

#### **E. Delivery / Publishing Features**
- **Cross-Platform**: Exporting to Windows (.exe), Mac (.app), Web (.html), Mobile (.apk).
- **Compression**: Optimizing file size for distribution.
- **Installer Creation**: Building a setup file for users.

---

### ‚ùñ 3. Types of Authoring Tools (Metaphors)

#### **A. Card- or Page-Based Tools**
- **Metaphor**: A book or a stack of index cards.
- **Structure**: The project consists of individual pages/cards.
- **Navigation**: "Go to Next Page", "Go to Previous Page".
- **Best For**: E-books, Simple Presentations, Information Kiosks.
- **Examples**:
    - **HyperCard** (The original).
    - **PowerPoint** (Slides are pages).
    - **ToolBook**.

#### **B. Icon-Based / Event-Driven Tools**
- **Metaphor**: A flowchart.
- **Structure**: You drag icons (representing events) onto a flow line.
- **Logic**: "Display Image" -> "Wait 3 Seconds" -> "Play Sound".
- **Best For**: Computer-Based Training (CBT), Complex logic flows.
- **Examples**:
    - **Adobe Authorware** (Industry standard for CBT).
    - **IconAuthor**.

#### **C. Time-Based Tools**
- **Metaphor**: A movie or timeline.
- **Structure**: Elements are placed on a timeline with layers.
- **Logic**: Things happen at specific frames or times (0:05s).
- **Best For**: Animation, Video-heavy projects, Games.
- **Examples**:
    - **Adobe Animate (formerly Flash)**.
    - **Adobe Director** (Shockwave).

#### **D. Object-Oriented Tools**
- **Metaphor**: Objects with properties and methods.
- **Structure**: Everything is an object that can interact with others.
- **Best For**: Games, Simulations.
- **Examples**: Unity, Unreal Engine.

---

### ‚ùñ Quick Revision Points
‚úì **Authoring**: Integration + Interactivity.
‚úì **Features**: Organization, Scripting, Publishing.
‚úì **Card-Based**: Pages (PowerPoint).
‚úì **Icon-Based**: Flowchart (Authorware).
‚úì **Time-Based**: Timeline (Flash).

## üî∑ 3. QUICKTIME

### ‚ùñ Overview
- **QuickTime** is a powerful multimedia framework developed by **Apple Inc.** in 1991.
- It is not just a "Player"; it is a comprehensive technology for handling video, sound, animation, graphics, text, interactivity, and music.
- It set the standard for digital video editing and playback.

---

### ‚ùñ 1. Architecture
- **File Format (.mov)**: A container format that can hold multiple tracks (Video, Audio, Text, Timecode).
- **Framework**: A set of APIs (Application Programming Interfaces) that allowed developers to use QuickTime features in their own apps (e.g., Adobe Premiere used QuickTime to export video).

---

### ‚ùñ 2. Key Features
- **Cross-Platform**: Available for both macOS and Windows (discontinued on Windows recently).
- **Versatility**: Could handle almost any media type (Images, Audio, Video, VR).
- **Streaming**: Supported real-time streaming of media over the internet.
- **QuickTime VR**: A special feature allowing users to view 360¬∞ panoramic photos and "look around" by dragging the mouse.
- **Compression**: Supported a wide range of codecs (Cinepak, Sorenson, H.264).

---

### ‚ùñ 3. Legacy and Impact
- QuickTime's file format (.mov) was the basis for the **MPEG-4 (MP4)** standard.
- It revolutionized digital video by making it accessible on personal computers.

---

### ‚ùñ Quick Revision Points
‚úì **Developer**: Apple.
‚úì **Format**: `.mov`.
‚úì **Feature**: Container for multiple tracks.
‚úì **VR**: QuickTime VR for panoramas.

## üî∑ 4. HYPERTEXT AND HYPERMEDIA

### ‚ùñ Overview
- **Hypertext** is the concept that powers the World Wide Web.
- It breaks the linear constraint of traditional text (start to finish) and allows **Non-Linear** navigation.
- **Hypermedia** expands this to include all media types.

---

### ‚ùñ 1. Definitions

#### **A. Hypertext**
- Text which contains links to other text.
- Coined by **Ted Nelson** in 1963.
- *Example:* Clicking a word in a Wikipedia article to jump to its definition.

#### **B. Hypermedia**
- An extension of hypertext that includes graphics, audio, video, and animation.
- *Example:* A webpage with text, a video player, and a clickable image map.

#### **C. Multimedia**
- Any combination of text, art, sound, animation, and video delivered by computer.
- *Difference*: Multimedia *can* be linear (a movie). Hypermedia *must* be interactive/linked.

---

### ‚ùñ 2. Elements of Hypertext

#### **A. Nodes**
- The fundamental unit of information.
- A node can be a document, a paragraph, an image, or a video clip.
- In the Web context, a **Webpage** is a node.

#### **B. Links (Hyperlinks)**
- The connection between two nodes.
- **Anchor**: The source of the link (usually blue, underlined text).
- **Destination**: The target node.
- **Types**:
    - **Internal**: Link to the same page.
    - **External**: Link to another website.

#### **C. Network**
- The structure formed by the collection of nodes and links.
- Can be simple (linear) or complex (web-like).

---

### ‚ùñ 3. Applications of Hypertext
- **World Wide Web (WWW)**: The largest implementation of hypertext (HTML = HyperText Markup Language).
- **Online Help Systems**: "Click here for troubleshooting" jumps to the relevant section.
- **E-Books / Digital Encyclopedias**: Cross-referencing topics.
- **Interactive Fiction**: "Choose Your Own Adventure" stories.
- **Software Documentation**: Linking error codes to solutions.

---

### ‚ùñ 4. Advantages vs Disadvantages

| **Advantages** | **Disadvantages** |
| :--- | :--- |
| **Non-Linear**: Freedom to explore. | **Disorientation**: "Lost in Hyperspace" (forgetting where you are). |
| **Depth**: Easy access to detailed info. | **Cognitive Load**: Too many choices can overwhelm the user. |
| **Updates**: Easy to add new nodes. | **Broken Links**: "404 Not Found" if a node is deleted. |
| **Cross-Reference**: Connecting related topics. | **Fragmented Reading**: Users may miss important context by skipping. |

---

### ‚ùñ Quick Revision Points
‚úì **Hypertext**: Linked Text.
‚úì **Hypermedia**: Linked Media.
‚úì **Node**: Information unit.
‚úì **Link**: Connection.
‚úì **Ted Nelson**: Coined the term.
‚úì **Issue**: Disorientation.
